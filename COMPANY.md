# NanoBanana Inc.

A startup providing simple API access to Google Gemini's image generation capabilities.

## Company Structure

### Board of Directors (Decision Makers)
- **Diego** - Major Shareholder, strategic direction
- **Claude (Orchestrator)** - Major Shareholder, operational coordination

### Executive Team

| Role | Responsibilities | Agent |
|------|------------------|-------|
| **CEO** | Business strategy, market positioning, pricing, partnerships | TBD |
| **CTO / Product Owner** | Technical architecture, product roadmap, feature prioritization | TBD |
| **Tech Lead** | Implementation, code quality, technical decisions | TBD |
| **QA Engineer** | Testing, code review, quality gates | TBD |

## Decision Flow

```
Directors (Diego + Claude)
    │
    ▼ Strategic decisions, funding, pivots
   CEO
    │
    ▼ Product direction, business priorities
   CTO/PO
    │
    ▼ Technical specs, architecture
   Tech Lead
    │
    ▼ Implementation
   QA Engineer ─── Review & approve
```

## Communication Protocol (Interim)

Until automated agent communication is set up:
1. Directors provide goals/decisions via text commands
2. Orchestrator (Claude) relays tasks to appropriate agent
3. Agents execute in terminal sessions
4. Results flow back through orchestrator
